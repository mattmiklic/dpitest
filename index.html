Sensible HiDPI Media Queries

These are the results of my media query test for determining how to serve high resolution assets to every browser. Here I’ve established a pixel ratio of 1.25x so that HiDPI assets are served to any display better than 1x. For more, read Making Sense of HiDPI Media Queries on my blog. Or, view the test.

A note on browser support: I’ve displayed a checkmark if a browser has complete support for that selector. An alert icon is shown where browser support is buggy or incomplete. For example: Chrome 30 for Windows understands min-resolution but does not report the correct dpi value for the screen.

Suggested Syntax

For each browser with HiDPI support, the most common method that it supports is highlighted in blue. For our media query, we combine the two rules that cover every browser, and add print so that high resolution assets are served when printing pages from your site.

@media print,
	(-webkit-min-device-pixel-ratio: 1.25),
	(min-resolution: 120dpi) {
		/* Styles go here */
	}
