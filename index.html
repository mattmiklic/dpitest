HiDPI Browser Support

These are the results of my media query test for determining how to serve high resolution assets to every browser. Here I’ve established a pixel ratio of 1.25x so that HiDPI assets are served to any display better than 1x. For more, read Making Sense of HiDPI Media Queries on my blog. Or, view the test.

A note on browser support: I’ve displayed a checkmark if a browser has complete support for that selector. An alert icon is shown where browser support is buggy or incomplete. For example: Opera 12.15 for OS X understands min-resolution but does not report the correct dpi value for the screen.

Suggested Syntax Back to Top

We combine the three rules that cover every browser, along with print so that even users without HiDPI displays get high resolution assets when printing.

@media print,
	(-o-min-device-pixel-ratio: 5/4),
	(-webkit-min-device-pixel-ratio: 1.25),
	(min-resolution: 120dpi) {
		/* Styles go here */
	}
